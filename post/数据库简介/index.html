数据库管理系统（Database Management System,简称DBMS）是为管理数据库而设计的软件系统，包括三大部分构成：

  1. 数据库文件集合：主要是一系列的数据文件，作用是存储数据
  2. 数据库服务端：主要负责对数据文件以及文件中的数据进行管理
  3. 数据库客户端：主要负责和服务端通信，向服务端传输数据或者从服务端获取数据

他们之间的关系：

  1. 数据库客户端通过 SQL 语句告诉服务端，客户端想要做什么
  2. 服务端和数据文件一般都在同一台机器上，直接可以读写数据文件

一般情况下，数据库分为关系型数据库和非关系型数据库

数据库排行榜: https://db-engines.com/en/ranking

  • 数据库（数据表的集合）

数据类型和约束

常见数据类型如下：

  • 整数：int, bit
  • 小数：decimal
  • 字符串：varchar, char
  • 日期时间：date, time, datetime
  • 枚举类型：enum

特别说明的类型如下：

  • decimal表示浮点数，如 decimal(5,2) 表示共存5位数，小数占 2 位
  • char 表示固定长度的字符串，如 char(3) ，填充 ‘ab’ 时就会补一个空格，储存 'ab '
  • varchar 表示可变长度的字符串，如 varchar(3) ，填充 ‘ab’ 时就会储存 ‘ab’
  • 对于图片、音频、视频等文件，不储存在数据库中，而是上传到某个服务器上，然后再表中储存这个文件的保存路径
  • 字符串 text 表示储存大文本，当字符大于4000时推荐使用，比如技术文档

数据约束：

约束本质上是对数据在数据类型限定的基础上添加的额外的要求

创建的约束如下：

  1. 主键 primary key ：物理上储存的顺序。MySQL 建议所有表的主键字段都叫 id ，类型为 int unsigned
  2. 非空 not null ：此字段不允许填写空值
  3. 唯一 unique ：此字段的值不允许重复
  4. 默认 default ：当不填写字段时，使用默认值；如果填写了，就以填写值为准
  5. 外键 foreign key : 对关系字段进行约束，当为关系字段填写值时，会到关联的表中查询此值是否存在，如果存在则填写成功，如果不存在则填写失败并抛出异常

var mysql = require("mysql")
var pool = mysql.createPool({
    host:"localhost",
    user:"root",
    password:"",
    database:"easystu"
})//数据库连接配置

function query(sql,callback){
    pool.getConnection(function(err,connection){
        connection.query(sql, function (err,rows) {
            callback(err,rows)
            connection.release()
        })
    })
}//对数据库进行增删改查操作的基础

exports.query = query

const db = require('./db.js');

async function handleDb(res,sql,errmsg){
    try{
        var ret = await new Promise((resolve, reject)=>{
            db.query(sql,(err, data)=>{
                if(err) {
                    reject(err);
                }
                resolve(data)
            });
        })
    }
    catch(err){
        res.send({errmsg:errmsg})
        return
    }
    
    return ret
}

module.exports = handleDb
